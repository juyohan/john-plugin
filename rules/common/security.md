# 보안 가이드라인

## 필수 보안 점검 사항

모든 커밋 전 확인 사항:
- [ ] 하드코딩된 비밀 정보(API 키, 비밀번호, 토큰)가 없는가?
- [ ] 모든 사용자 입력이 검증되는가?
- [ ] SQL 인젝션 방지가 되어 있는가? (파라미터화된 쿼리 사용)
- [ ] XSS 방지가 되어 있는가? (HTML 정제)
- [ ] CSRF 방지가 활성화되어 있는가?
- [ ] 인증 및 인가가 확인되었는가?
- [ ] 모든 엔드포인트에 처리량 제한(Rate limiting)이 적용되었는가?
- [ ] 에러 메시지에 민감한 정보가 유출되지 않는가?

## 비밀 정보(Secret) 관리

- 소스 코드에 비밀 정보를 절대 하드코딩하지 마십시오.
- 항상 환경 변수나 비밀 관리 도구(Secret Manager)를 사용하십시오.
- 시작 시 필수 비밀 정보가 존재하는지 검증하십시오.
- 노출되었을 가능성이 있는 모든 비밀 정보는 즉시 교체(Rotate)하십시오.

## 보안 대응 프로토콜

보안 이슈 발견 시:
1. 즉시 작업을 중단하십시오.
2. **security-reviewer** 에이전트를 사용하십시오.
3. 작업을 계속하기 전에 '치명적(CRITICAL)' 이슈를 먼저 해결하십시오.
4. 노출된 모든 비밀 정보를 교체하십시오.
5. 유사한 이슈가 있는지 전체 코드베이스를 검토하십시오.
