# Brainstorm: TDD SKILL.md 리팩토링

**날짜:** 2026-05-13
**대상 파일:** `skills/tdd/SKILL.md`

---

## 문제

현재 TDD 스킬(412줄)이 너무 길고, Mock 위주의 테스트 패턴이 통합 테스트보다 강조되어 있다.
- Supabase/Redis/OpenAI Mock 코드 블록 3개가 전체 분량의 30%를 차지
- 실제 의존성을 직접 띄우는 통합 테스트 방법론이 없음
- plan/brainstorm 수준의 간결한 지침이 되어야 함

---

## 요구사항

### R1. 길이 단축
- 목표: 120~150줄 (현재 412줄)
- 삭제 대상: Mock 코드 블록 3개, 모범 사례 10개 목록, 성공 지표, 지속적 테스팅 섹션, 중복 코드 예시

### R2. 통합 테스트 우선 원칙 명시
- 자신이 소유한 서비스(DB, 캐시, API 서버)는 실제로 띄워서 테스트
- "실 의존성 기반 통합 테스트 → Mock은 외부 써드파티 경계만" 원칙 명문화
- 테스트 환경 구성 방식 간략 서술 (docker-compose, 테스트 DB 등)

### R3. Mock 허용 범위 명확화
- 완전 제거 아님 — 외부 써드파티 API(결제, SMS, 지도 등) 경계에서만 허용
- Mock 섹션을 독립 코드 블록에서 "허용 범위" 한 단락으로 축소

### R4. 핵심 구조 유지
- TDD 7단계 워크플로우 (압축)
- 테스트 유형: 단위 / 통합 / E2E
- AAA 패턴, 파일 구조, 커버리지 80% 기준

---

## 결정된 구조

재작성 후 `skills/tdd/SKILL.md` 섹션 순서:

```
# TDD 워크플로우
## 활성화 시점           (~5줄)
## 테스트 유형 + 우선순위  (~15줄, 통합 테스트 우선 원칙 포함)
## Mock 허용 범위        (~5줄)
## TDD 7단계            (~20줄, 압축)
## 테스트 파일 구조       (~15줄)
## 커버리지 기준          (~5줄)
## 주의사항              (~10줄, 취약한 셀렉터/구현 세부사항 테스트 금지)
```
