# 성능 최적화

## 모델 선택 전략

**Haiku 4.5** (Sonnet의 90% 성능, 비용 3배 절감):
- 빈번하게 호출되는 경량 에이전트
- 페어 프로그래밍 및 코드 생성
- 다중 에이전트 시스템 내의 워커(Worker) 에이전트

**Sonnet 4.6** (최고의 코딩 모델):
- 메인 개발 작업
- 다중 에이전트 워크플로우 오케스트레이션
- 복잡한 코딩 태스크

**Opus 4.7** (가장 깊은 추론 능력):
- 복잡한 아키텍처 결정
- 최대 수준의 추론이 필요한 작업
- 조사 및 분석 태스크

## 컨텍스트 윈도우 관리

컨텍스트 윈도우의 마지막 20%는 다음 작업에 사용하지 마십시오:
- 대규모 리팩토링
- 여러 파일에 걸친 기능 구현
- 복잡한 상호작용 디버깅

컨텍스트 민감도가 낮은 작업들:
- 단일 파일 편집
- 독립적인 유틸리티 생성
- 문서 업데이트
- 단순한 버그 수정

## 확장 사고(Extended Thinking) + 플랜 모드

확장 사고는 기본적으로 활성화되어 있으며, 내부 추론을 위해 최대 31,999 토큰을 예약합니다.

확장 사고 제어 방법:
- **토글**: Option+T (macOS) / Alt+T (Windows/Linux)
- **설정**: `~/.claude/settings.json`에서 `alwaysThinkingEnabled` 설정
- **예산 제한**: `export MAX_THINKING_TOKENS=10000`
- **상세 모드**: Ctrl+O를 눌러 사고 과정 출력 확인

깊은 추론이 필요한 복잡한 태스크의 경우:
1. 확장 사고가 활성화되어 있는지 확인하십시오 (기본 활성).
2. 구조적 접근을 위해 **플랜 모드(Plan Mode)**를 활성화하십시오.
3. 철저한 분석을 위해 여러 단계의 비판(critique) 과정을 거치십시오.
4. 다양한 관점을 위해 역할을 분담한 서브 에이전트들을 활용하십시오.

## 빌드 문제 해결

빌드가 실패할 경우:
1. **build-error-resolver** 에이전트를 활용하십시오.
2. 에러 메시지를 분석하십시오.
3. 점진적으로 수정하십시오.
4. 각 수정 단계마다 검증하십시오.
