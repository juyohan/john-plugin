# 전략 템플릿 (Strategy Template)

인터뷰가 완료된 후 `SKILL.md`에 의해 로드됩니다. 수집된 답변을 사용하여 템플릿을 채우고 `STRATEGY.md`에 작성하십시오.

## 작성 규칙

- 가능한 경우 사용자의 언어를 그대로 사용하십시오. 일반적인 기획자용 용어(PM-speak)로 바꾸지 마십시오.
- 각 섹션은 콤팩트하게 유지하십시오. 전체 문서를 5분 이내에 읽을 수 있어야 합니다.
- 섹션 순서는 고정되어 있습니다. 새로운 최상위 섹션을 추가하지 마십시오.
- 선택 섹션: 사용되지 않은 경우 섹션 전체를 삭제하십시오. 빈 헤더를 남겨두지 마십시오.
- YAML 프론트매터의 `last_updated`를 오늘 날짜(YYYY-MM-DD)로 설정하십시오. 본문에서 날짜를 중복 기재하지 마십시오.
- 프론트매터의 `name`을 제품 또는 이니셔티브 이름(H1 제목과 동일한 값)으로 설정하십시오.

## 템플릿

아래 블록은 작성될 실제 파일 내용입니다. 모든 `{{placeholder}}`를 수집된 답변으로 교체하십시오. 답변되지 않은 선택 섹션의 자리는 삭제하십시오.

~~~markdown
---
name: {{product_name}}
last_updated: {{YYYY-MM-DD}}
---

# {{product_name}} 전략 (Strategy)

## 대상 문제 (Target problem)

{{1-2문장의 진단. 사용자의 상황과 문제를 어렵게 만드는 핵심(crux)을 명시합니다. 솔루션 언어를 포함하지 않습니다.}}

## 접근 방식 (Our approach)

{{1-2문장의 가이드 원칙. 대상 문제를 해결 가능하게 만들기 위해 이 제품이 약속하거나 집중하는 지점입니다.}}

## 대상 사용자 (Who it's for)

**주요 사용자:** {{페르소나 이름}} - {{한 문장의 JTBD, 예: "그들은 {{product_name}}을 사용하여 ~하려 합니다."}}

<!-- 정말 필요한 경우에만 위 블록을 복제하여 추가 페르소나를 기재하십시오. 적을수록 좋습니다. -->

## 주요 지표 (Key metrics)

- **{{지표 1 이름}}** - {{한 줄 정의; 측정 위치}}
- **{{지표 2 이름}}** - {{...}}
- **{{지표 3 이름}}** - {{...}}

<!-- 총 3-5개. 5개까지만 작성하십시오. -->

## 트랙 (Tracks)

### {{트랙 1 이름}}

{{한 줄 요약: 이 트랙이 무엇인지 - 기능 목록이 아닌 투자 영역입니다.}}

_접근 방식에 기여하는 이유:_ {{한 줄 요약}}

<!-- 총 2-4개의 트랙을 위해 위 블록을 복제하십시오. 4개를 넘는다면 관련 트랙들을 하나로 묶으십시오. -->

## 마일스톤 (Milestones)

- **{{YYYY-MM-DD}}** - {{마일스톤 내용}}

<!-- 선택 사항. 사용하지 않으면 섹션을 삭제하십시오. 출시, 투자 유치, 컨퍼런스 등 외부 노출 마일스톤만 작성하십시오. -->

## 수행하지 않는 작업 (Not working on)

- {{항목당 한 줄}}

<!-- 선택 사항. 사용하지 않으면 섹션을 삭제하십시오. 팀이 계속 유혹을 느끼는 항목들만 작성하십시오. -->

## 마케팅 (Marketing)

**한 줄 요약 (One-liner):** {{한 문장의 피치}}

**핵심 메시지 (Key message):** {{필요한 경우 2-3행 내외}}

<!-- 선택 사항. 사용하지 않으면 섹션을 삭제하십시오. -->
~~~

## 작성 후 체크리스트

파일을 작성하기 전 다음 사항을 확인하십시오:

- [ ] 상단에 `name`과 `last_updated` 키가 포함된 프론트매터가 존재합니다.
- [ ] `last_updated`에 오늘 날짜가 ISO 형식(YYYY-MM-DD)으로 기재되었습니다.
- [ ] 트랙 섹션(각 트랙당 하나의 블록)을 제외한 나머지 섹션은 4문장을 넘지 않습니다.
- [ ] 남아 있는 플레이스홀더(`{{...}}`)가 없습니다.
- [ ] 내용이 없는 선택 섹션은 빈 채로 두지 않고 완전히 삭제되었습니다.
- [ ] 지표의 개수는 3개에서 5개 사이입니다. 트랙의 개수는 2개에서 4개 사이입니다.
- [ ] '대상 문제'와 '접근 방식'이 서로 연결되어 있습니다 - 하나가 다른 하나에 명확히 대응합니다.
