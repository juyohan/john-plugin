# 테스트 요구사항

## 최소 테스트 커버리지: 80%

테스트 유형 (모두 필수):
1. **단위 테스트 (Unit Tests)** - 개별 함수, 유틸리티, 컴포넌트
2. **통합 테스트 (Integration Tests)** - API 엔드포인트, 데이터베이스 작업
3. **E2E 테스트 (E2E Tests)** - 핵심 사용자 흐름 (언어별 프레임워크 선택)

## 테스트 주도 개발 (TDD)

필수 워크플로우:
1. 테스트를 먼저 작성합니다 (RED).
2. 테스트를 실행하여 실패하는지 확인합니다.
3. 테스트를 통과할 정도의 최소한의 코드를 구현합니다 (GREEN).
4. 테스트를 실행하여 통과하는지 확인합니다.
5. 리팩토링을 수행합니다 (IMPROVE).
6. 커버리지를 확인합니다 (80% 이상).

## 테스트 실패 해결

1. **tdd-guide** 에이전트를 활용하십시오.
2. 테스트 격리(isolation) 여부를 확인하십시오.
3. 모킹(mock)이 올바른지 검증하십시오.
4. 테스트 코드가 잘못된 것이 아니라면, 테스트가 아닌 구현 코드를 수정하십시오.

## 에이전트 지원

- **tdd-guide** - 새로운 기능 개발 시 선제적으로 사용하며, 테스트 우선 작성을 강제합니다.

## 테스트 구조 (AAA 패턴)

테스트 작성 시 Arrange-Act-Assert 구조를 선호합니다:

```typescript
test('유사도를 올바르게 계산한다', () => {
  // Arrange (준비)
  const vector1 = [1, 0, 0]
  const vector2 = [0, 1, 0]

  // Act (실행)
  const similarity = calculateCosineSimilarity(vector1, vector2)

  // Assert (검증)
  expect(similarity).toBe(0)
})
```
