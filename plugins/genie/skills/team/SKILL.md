# genie:team — 전문가 팀 오케스트레이션

리드(메인 Claude)가 3명의 전문가를 조율하여 중요한 결정에 대한 다각도 검토를 수행합니다.

## 팀 구성

| 역할 | 에이전트 | 담당 |
|------|----------|------|
| ECC 전문가 | `genie-ecc-expert` | ECC 패턴·컨벤션·호환성 관점 |
| CE 전문가 | `genie-ce-expert` | CE 워크플로우·스킬 설계 관점 |
| 검증자 | `genie-reviewer` | 두 의견 취합 후 최종 비판적 검증 |
| **리드** | **메인 Claude** | **최종 판단 및 실행** |

---

## 워크플로우

### Phase 1 — 주제 파악

`$ARGUMENTS`가 있으면 그것을 검토 주제로 사용.
없으면 현재 컨텍스트(최근 변경 사항, 논의 중인 제안)를 주제로 삼는다.

주제를 한 문장으로 정리한 뒤 Phase 2로 진행.

---

### Phase 2 — 병렬 전문가 분석

ECC 전문가와 CE 전문가를 **동시에** 호출한다.

```
Agent(genie-ecc-expert): "<주제> — ECC 관점에서 분석해주세요"
Agent(genie-ce-expert):  "<주제> — CE 관점에서 분석해주세요"
```

두 에이전트가 반환한 의견을 그대로 보존한다. 이 단계에서 리드는 판단하지 않는다.

---

### Phase 3 — 검증자 검토

두 전문가 의견을 검증자에게 전달한다.

```
Agent(genie-reviewer):
  "다음 두 전문가 의견을 검증해주세요.

  [ECC Expert 의견]
  <Phase 2 결과>

  [CE Expert 의견]
  <Phase 2 결과>"
```

검증자의 종합 판정(✅/⚠️/❌)과 우려 사항을 수신한다.

---

### Phase 4 — 리드 최종 판단

리드(메인 Claude)가 세 의견을 종합하여 최종 결론을 내린다.

**출력 형식:**

```
## 팀 검토 결과

**주제**: <한 줄 요약>

### ECC 관점
<핵심 요약 2-3줄>

### CE 관점
<핵심 요약 2-3줄>

### 검증자 판정
<✅/⚠️/❌> + 주요 우려 사항

### 리드 결론
<최종 결정 및 근거>

### 다음 액션
- [ ] <구체적 실행 항목>
```

---

## 언제 사용하는가

- `AGENTS.md`, `SKILL.md` 등 핵심 파일 변경 전
- ECC와 CE 두 프로젝트에 영향을 미치는 설계 결정
- 컨벤션 충돌이 예상되는 제안
- 확신이 없는 중요한 방향 결정

## 언제 사용하지 않는가

- 단순 파일 편집, 버그 수정
- 이미 명확한 작업 (바로 실행)
- 빠른 답변이 필요한 사소한 질문
