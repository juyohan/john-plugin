# 코드 리뷰 표준

## 목적

코드 리뷰는 코드가 머지되기 전에 품질, 보안 및 유지보수성을 보장하기 위해 수행됩니다. 이 규칙은 코드 리뷰를 언제, 어떻게 수행할지 정의합니다.

## 리뷰 시점

**필수 리뷰 트리거:**

- 코드 작성 또는 수정 후
- 공유 브랜치에 커밋하기 전
- 보안에 민감한 코드 변경 시 (인증, 결제, 사용자 데이터 등)
- 아키텍처 변경 시
- 풀 리퀘스트(PR) 머지 전

**리뷰 전 요구사항:**

리뷰를 요청하기 전에 다음 사항을 확인하십시오:

- 모든 자동화된 검사(CI/CD) 통과 여부
- 머지 충돌(Merge conflicts) 해결 여부
- 대상 브랜치(Target branch)와 최신 상태 유지 여부

## 리뷰 체크리스트

코드를 완료로 표시하기 전 다음을 확인하십시오:

- [ ] 코드가 가독성이 좋고 이름이 적절한가?
- [ ] 함수가 하나의 기능에 집중되어 있는가? (<50라인)
- [ ] 파일의 응집도가 높은가? (<800라인)
- [ ] 깊은 중첩이 없는가? (>4단계 지양)
- [ ] 에러가 명시적으로 처리되었는가?
- [ ] 하드코딩된 비밀 정보나 자격 증명이 없는가?
- [ ] `console.log`나 디버그 문이 남아있지 않은가?
- [ ] 새로운 기능에 대한 테스트가 존재하는가?
- [ ] 테스트 커버리지가 최소 80%를 충족하는가?

## 보안 리뷰 트리거

**다음의 경우 작업을 중단하고 security-reviewer 에이전트를 사용하십시오:**

- 인증(Authentication) 또는 인가(Authorization) 코드
- 사용자 입력 처리
- 데이터베이스 쿼리
- 파일 시스템 작업
- 외부 API 호출
- 암호화 작업
- 결제 또는 금융 관련 코드

## 리뷰 심각도 수준 (Severity Levels)

| 수준 | 의미 | 조치 |
|-------|---------|--------|
| 치명적 (CRITICAL) | 보안 취약점 또는 데이터 손실 위험 | **차단 (BLOCK)** - 머지 전 반드시 수정 |
| 높음 (HIGH) | 버그 또는 심각한 품질 이슈 | **경고 (WARN)** - 머지 전 수정 권장 |
| 중간 (MEDIUM) | 유지보수 관련 우려 사항 | **정보 (INFO)** - 수정 고려 |
| 낮음 (LOW) | 스타일 또는 사소한 제안 | **참고 (NOTE)** - 선택 사항 |

## 에이전트 활용

코드 리뷰를 위해 다음 에이전트들을 활용하십시오:

| 에이전트 | 용도 |
|-------|---------|
| **code-reviewer** | 일반적인 코드 품질, 패턴, 베스트 프랙티스 |
| **security-reviewer** | 보안 취약점, OWASP Top 10 |
| **typescript-reviewer** | TypeScript/JavaScript 전용 이슈 |
| **python-reviewer** | Python 전용 이슈 |
| **go-reviewer** | Go 전용 이슈 |
| **rust-reviewer** | Rust 전용 이슈 |
| **java-reviewer** | Java/Spring Boot 전용 이슈 |
| **kotlin-reviewer** | Kotlin/Android/KMP 전용 이슈 |
| **swift-reviewer** | Swift/iOS 전용 이슈 |

> **이 프로젝트에서**: `/genie:review` 커맨드가 언어별 리뷰어를 자동 선택합니다. Genie 커맨드 별칭(`ts`, `py`, `go`, `security`, `review`)은 `commands/` 디렉토리를 통해 위 에이전트를 호출합니다.

## 리뷰 워크플로우

```
1. git diff를 실행하여 변경 사항 파악
2. 보안 체크리스트 우선 확인
3. 코드 품질 체크리스트 검토
4. 관련 테스트 실행
5. 커버리지 80% 이상 확인
6. 상세 리뷰를 위해 적절한 에이전트 활용
```

## 주요 점검 사항

### 보안

- 하드코딩된 자격 증명 (API 키, 비밀번호, 토큰)
- SQL 인젝션 (쿼리 내 문자열 결합)
- XSS 취약점 (이스케이프되지 않은 사용자 입력)
- 경로 조작 (정제되지 않은 파일 경로)
- CSRF 방지 누락
- 인증 우회

### 코드 품질

- 거대 함수 (50라인 초과) - 기능별로 분리
- 거대 파일 (800라인 초과) - 모듈 추출
- 깊은 중첩 (4단계 초과) - 조기 반환 사용
- 에러 처리 누락 - 명시적으로 처리
- 변경(Mutation) 패턴 - 불변 연산 선호
- 테스트 누락 - 테스트 커버리지 추가

### 성능

- N+1 쿼리 - JOIN 또는 배치 처리 사용
- 페이지네이션 누락 - 쿼리에 LIMIT 추가
- 제한 없는 쿼리 - 제약 조건 추가
- 캐싱 누락 - 비용이 큰 작업 캐싱 처리

## 승인 기준

- **승인 (Approve)**: 치명적(CRITICAL) 또는 높음(HIGH) 이슈 없음
- **경고 (Warning)**: 높음(HIGH) 이슈만 존재 (주의하여 머지 가능)
- **차단 (Block)**: 치명적(CRITICAL) 이슈 발견
