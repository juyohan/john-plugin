# genie:setup — 프로젝트 온보딩 스킬

**날짜:** 2026-05-13
**상태:** 브레인스토밍 완료 → 플랜 대기

---

## 개요

완전히 빈 프로젝트(아무것도 없음)에 genie 플러그인을 설치한 직후 실행하는 온보딩 스킬.
프로젝트를 자동 스캔하여 스택을 감지하고, Claude가 코딩 시 참고할 수 있는 프로젝트 인텔리전스 문서를 생성한다.

---

## Actors (A-IDs)

| ID | Actor | 설명 |
|----|-------|------|
| A1 | 새 사용자 | genie 플러그인을 방금 설치하고 프로젝트 셋업을 처음 시작하는 사람 |
| A2 | Claude | 생성된 문서를 이후 작업에서 컨텍스트로 참조하는 에이전트 |

---

## Requirements (R-IDs)

| ID | 요구사항 |
|----|---------|
| R1 | 실행 시 프로젝트 파일을 자동 스캔하여 언어, 프레임워크, 패키지 매니저, 테스트 프레임워크, 디렉토리 구조를 감지한다 |
| R2 | 자동 스캔으로 판단할 수 없는 항목(팀 컨벤션, 선호 패턴 등)에 대해서만 사용자에게 질문을 던진다 |
| R3 | 프로젝트 루트에 `CLAUDE.md`를 생성한다 |
| R4 | `docs/conventions.md`를 생성한다 — 네이밍 규칙, 코드 스타일, 선호 패턴 |
| R5 | `docs/architecture.md`를 생성한다 — 디렉토리 구조, 레이어 구성, 주요 설계 결정 |
| R6 | `docs/setup.md`를 생성한다 — 실행/테스트/빌드 방법 |
| R7 | 글로벌 `~/.claude/rules/`는 건드리지 않는다. 모든 산출물은 프로젝트 로컬에만 위치한다 |
| R8 | 생성된 파일들은 프로젝트와 함께 이동 가능하며, 독립적으로 재사용/공유할 수 있다 |
| R9 | `CLAUDE.md`는 프로젝트 이름과 각 참조 파일이 어떤 내용인지 설명하는 이름을 포함하고, `@docs/` 참조로 상세 문서를 연결한다 |
| R10 | `CLAUDE.md`가 참조하는 파일은 상시 참조 대상(conventions, architecture, setup)으로만 한정한다. brainstorms/, plans/ 등 워크플로우 산출물은 참조하지 않는다 |
| R11 | 실행 시 `CLAUDE.md` 또는 `docs/` 파일이 이미 존재하면, 덮어쓰기 또는 병합 중 어떻게 처리할지 사용자에게 확인한다 |

## Key Flows (F-IDs)

| ID | 플로우 |
|----|--------|
| F1 | 사용자가 `/genie:setup` 실행 → 프로젝트 파일 스캔 → 스택 감지 결과 출력 |
| F2 | 기존 파일 존재 감지 시 → 덮어쓰기 / 병합 여부 사용자에게 확인 |
| F3 | 불확실한 항목 발견 시 → 질문 1개씩 순서대로 던짐 → 사용자 답변 수집 |
| F4-1 | 수집된 정보 기반으로 `CLAUDE.md` + `docs/` 3개 파일 생성 |
| F5 | 완료 후 생성된 파일 목록과 "다음 단계: `/genie:brainstorm`" 안내 출력 |

| AE4 | 모노레포 (백엔드+프론트엔드 혼재) | 멀티 스택 감지 → 각 레이어별로 컨벤션 분리 기록 |

## 생성 파일 구조

```
프로젝트 루트/
├── CLAUDE.md                  ← 프로젝트 이름 + 각 docs 파일 설명 + @docs/ 참조
└── docs/
    ├── conventions.md         ← 네이밍, 코드 스타일, 선호 패턴 (상세)
    ├── architecture.md        ← 디렉토리 구조, 레이어, 설계 결정 (상세)
    ├── setup.md               ← 실행/테스트/빌드 방법 (상세)
    ├── brainstorms/           ← 워크플로우 기록 (CLAUDE.md 참조 제외)
    ├── plans/                 ← 워크플로우 기록 (CLAUDE.md 참조 제외)
    └── work/                  ← 워크플로우 기록 (CLAUDE.md 참조 제외)
```

### CLAUDE.md 구조 패턴

```markdown
# {프로젝트 이름}

## 코드 컨벤션 (docs/conventions.md)
네이밍 규칙, 코드 스타일, 선호 패턴
@docs/conventions.md

## 아키텍처 (docs/architecture.md)
디렉토리 구조, 레이어 구성, 주요 설계 결정
@docs/architecture.md

## 개발 환경 (docs/setup.md)
실행, 테스트, 빌드 방법
@docs/setup.md
```
